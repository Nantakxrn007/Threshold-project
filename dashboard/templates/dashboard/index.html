🔴 TH1 — Sliding Percentile

ใช้ percentile ของ error ใน window ก่อนหน้าเป็น threshold — สูง = conservative (จับน้อย), ต่ำ = sensitive (จับมาก)

Sliding = คำนวณจากข้อมูลก่อนหน้า N จุด ไม่ใช้ future data
99.0
P99 = threshold ที่ 99% ของ normal error อยู่ต่ำกว่า
100
จำนวน data points ที่มองย้อนหลัง — ใหญ่ = stable แต่ตอบสนองช้า
10
คำนวณ threshold ใหม่ทุก N จุด — 1 = smooth สุด แต่ช้ากว่า
🔵 TH2 — Sliding μ + α·σ

threshold = mean + α × std ของ error ใน window ก่อนหน้า — ปรับตามความผันผวนของสัญญาณอัตโนมัติ

3.5
สูง = conservative (ต้องเกินค่าเฉลี่ยไปมากๆ ถึงจะ flag)
80
ขนาด sliding window สำหรับคำนวณ μ และ σ
50
คำนวณ μ/σ ใหม่ทุก N จุด — น้อย = ตามทัน แต่ช้ากว่า
🟢 TH3 — Adaptive-z

ปรับ z-score multiplier อัตโนมัติตาม anomaly density ที่ตรวจเจอใน window — ไม่ต้องตั้ง α เอง

2.0
z ต่ำสุดที่ยอมให้ — ป้องกันไม่ให้ threshold ต่ำเกินไป
10.0
z สูงสุดที่ยอมให้ — ป้องกันไม่ให้ threshold สูงเกินจนจับไม่ได้
80
10
🟣 TH4 — Entropy-lock

lock window size เมื่อ entropy ของ error distribution นิ่งพอ — ไม่ให้ window ขยายออกไปเรื่อยๆ จน threshold หลวม

3.5
เหมือน TH2 — μ + α·σ ของ window ที่ lock
150
window ขยายได้สูงสุด N จุด ก่อน entropy lock ทำงาน
5
ต้องเห็น entropy นิ่ง N ครั้งติดกันก่อน lock — มากขึ้น = lock ช้าลง แต่ stable กว่า
0.95
ถ้า ΔH < (1-θ)×ΔH₀ ถือว่า entropy นิ่ง — สูงขึ้น = lock ง่ายขึ้น
